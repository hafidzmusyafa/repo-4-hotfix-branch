# \## Strategi

# 

# Strategi yang digunakan adalah Hotfix Branch.

# 

# Hotfix branch digunakan untuk memperbaiki bug pada versi rilis. Branch hotfix dibuat dari master, lalu hasil perbaikan disinkronkan kembali ke master dan develop.

# 

# \## Branch

# 

# \- master: branch rilis/stabil

# \- develop: branch pengembangan

# \- hotfix/fix-login-validation: branch perbaikan bug login

# 

# \## Command yang didemokan

# 

# \- git checkout

# \- git pull

# \- git status

# \- git add

# \- git commit

# \- git push

# \- git revert
